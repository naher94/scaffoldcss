Scaffold CSS

A lightweight CSS layout library — utility classes and SCSS mixins for Grid and Flexbox, with zero dependencies and sensible defaults.

Work in progress — spec draft

Philosophy

Scaffold stays close to native CSS Grid rather than abstracting it away. The goal is to fill the gaps where Grid is verbose or repetitive, not to reinvent it. If you know CSS Grid, you already know most of Scaffold.

Mixins are the primitive layer. Every layout concept — grid, span, flex, container, gap — is available as a SCSS mixin first. Utility classes are generated output from those same mixins, so both approaches produce identical CSS. Use whichever fits your workflow.

Everything is customizable via CSS custom properties. Classes are the convenience layer; custom properties are the real configuration surface.

All classes are prefixed with scaffold- to avoid conflicts with other libraries or existing project CSS. Custom properties use the same --scaffold- namespace, so there's one consistent naming system throughout.

Grid or flex?

The deciding question is: does the layout structure exist before the content, or does the content determine the size?

If you're defining the structure first — a 12-column page layout, a fixed card grid — that's Grid. The tracks exist regardless of what goes in them. If the content is driving its own size and you're just controlling how items flow and align — a nav bar, a tag list, a button group — that's Flex.

Installation

Using SCSS

Copy the _sass/scaffold/ folder into your project's _sass/ directory, then import the entry point:

// your main stylesheet
@use 'scaffold/scaffold';

Using compiled CSS

Link the compiled CSS directly if you don't need the SCSS mixins:

<link rel="stylesheet" href="dist/scaffold.css">

What's included

Scaffold ships two artifacts: the compiled dist/scaffold.css for drop-in use, and the raw _sass/scaffold/ source for projects that want to use the mixins or customize the token defaults. Use whichever fits your setup — they produce the same utility classes.

If you use the SCSS route with mixins only and never reference utility classes in your HTML, the compiled output will be significantly smaller since only the mixins you actually call get included. The compiled dist/scaffold.css includes all utility classes regardless of usage — there is no tree-shaking.

Grid

The .scaffold-grid class sets display: grid and a 12-column layout in one step. The equivalent grid() mixin produces the same output for use inside your own SCSS rules.

<div class="scaffold-grid">
  <div class="scaffold-grid-span-half">Left</div>
  <div class="scaffold-grid-span-half">Right</div>
</div>
.my-layout {
  @include scaffold.grid();
}

.my-layout__item {
  @include scaffold.span(half);
}
Preview — two halves

Overriding column count

The --scaffold-cols custom property lets you change the column structure on a per-instance basis. The mixin also accepts a $cols argument.

<!-- Via custom property on the element -->
<div class="scaffold-grid" style="--scaffold-cols: repeat(8, 1fr)">...</div>

// Via mixin argument
.my-layout {
  @include scaffold.grid($cols: repeat(8, 1fr));
}

Overrides are scoped to that element only. Every .scaffold-grid declares --scaffold-cols locally, so nested grids always reset to 12 columns and are unaffected by a parent's column override. Span utilities remain safe to use inside nested grids.

Grid span utilities

Grid span classes define how many of the 12 columns a child element occupies. The span() mixin accepts the same semantic names as the utility classes. The grid-span naming makes clear these are only meaningful inside a .scaffold-grid container.

Important Span utilities assume a 12-column grid. They output fixed grid-column: span N values. If you override --scaffold-cols on a grid, span utilities on that grid's direct children will no longer represent their named fractions — use explicit grid-column values instead. Nested .scaffold-grid elements are unaffected and always reset to 12 columns.

Overflow wrapping

If spans in a row add up to more than 12 columns, CSS Grid will automatically wrap the overflowing item to a new row. This is intentional default behavior — for example, two .scaffold-grid-span-two-thirds elements side by side will stack rather than overflow their container.

Span utilities in flex contexts

All span utilities output grid-specific grid-column values and have no effect inside a .scaffold-flex container. Span utilities are only meaningful inside .scaffold-grid.

When to skip span utilities altogether

If all children in a grid are the same width, you can skip span utilities entirely and let the grid do the work. Instead of declaring a span on every child, change the column count on the container — children will flow into equal tracks automatically with no extra markup needed.

<!-- Without: span class on every child -->
<div class="scaffold-grid">
  <div class="scaffold-grid-span-third">...</div>
  <div class="scaffold-grid-span-third">...</div>
  <div class="scaffold-grid-span-third">...</div>
</div>

<!-- With: column count on the container, children flow automatically -->
<div class="scaffold-grid" style="--scaffold-cols: repeat(3, 1fr)">
  <div>...</div>
  <div>...</div>
  <div>...</div>
</div>

Flex

A layout layer for cases where Grid isn't the right tool — primarily single-axis layouts, component-level alignment, and inline groupings. Available as both utility classes and the flex() mixin.

<div class="scaffold-flex scaffold-flex-col scaffold-gap-sm">
  <div>Item one</div>
  <div>Item two</div>
</div>
.my-stack {
  @include scaffold.flex($direction: column);
  gap: var(--scaffold-gap-sm);
}

Usage note

Alignment classes are container-level properties and only take effect on .scaffold-flex or .scaffold-grid elements. Applying them to a regular block element will have no visual effect.

Proportional flex items

.scaffold-flex-intrinsic is a child utility for proportional side-by-side layouts where items should share the same height and fill the full row width. It works by setting flex to a custom property representing the item's aspect ratio — wider items take more space, narrower items take less, and all items align to the same height.

This class is only meaningful inside a .scaffold-flex container — the naming makes that clear.

<!-- Portrait + landscape, equal height, no magic numbers in the class -->
<div class="scaffold-flex">
  <div class="scaffold-flex-intrinsic" style="--aspect-ratio: calc(600/900)">
    <img src="portrait.jpg">
  </div>
  <div class="scaffold-flex-intrinsic" style="--aspect-ratio: calc(1200/800)">
    <img src="landscape.jpg">
  </div>
</div>

The --aspect-ratio value is width ÷ height. Using calc() keeps the original image dimensions readable rather than requiring you to pre-compute the decimal.

If --aspect-ratio is not set, the item falls back to flex: 1 — equal widths, no breakage.

Child content width

This class is commonly used for items like side-by-side images to create a proportional layout. For this to work correctly, the child content — the image — needs to be set with width: 100% so it fills the flex item rather than sitting at its natural size.

img {
  width: 100%;
  display: block;
}

Stacking on mobile

Use the existing responsive flex direction classes on the parent to stack items on small screens:

<div class="scaffold-flex scaffold-flex-col md:scaffold-flex">
  <div class="scaffold-flex-intrinsic" style="--aspect-ratio: calc(600/900)">
    <img src="portrait.jpg">
  </div>
  <div class="scaffold-flex-intrinsic" style="--aspect-ratio: calc(1200/800)">
    <img src="landscape.jpg">
  </div>
</div>

Container

A max-width wrapper for centering content within a page. Available as .scaffold-container or the container() mixin. Configurable via custom properties so each project can set its own content width without modifying the library.

<div class="scaffold-container">
  <div class="scaffold-grid">...</div>
</div>
.page-wrapper {
  @include scaffold.container();
}

// With a custom max-width
.page-wrapper--wide {
  @include scaffold.container($width: 1440px);
}

Full-width variant

.scaffold-container-full is a separate class for layouts that should span the entire viewport width while still keeping the side padding. Use it when you want edge-to-edge content without a max-width cap.

<div class="scaffold-container-full">
  <div class="scaffold-grid">...</div>
</div>

Configuring width and padding

Container padding uses two tokens — a smaller value for mobile and a larger one for wider viewports. Both are overridable independently.

:root {
  --scaffold-container-width:      1440px;
  --scaffold-container-padding-sm: 1rem;   /* mobile */
  --scaffold-container-padding:    2rem;   /* md and up */
}

Gap

Gap utility classes map to three t-shirt sizes backed by custom properties. The gap() mixin accepts the same size tokens.

<div class="scaffold-grid scaffold-gap-lg">...</div>
Class Default value Custom property
.scaffold-gap-sm0.5rem--scaffold-gap-sm
.scaffold-gap-md1rem--scaffold-gap-md
.scaffold-gap-lg2rem--scaffold-gap-lg

Independent row and column gap

By default, row and column gap are the same value. To control them independently, scope --scaffold-row-gap or --scaffold-col-gap directly on the element — no extra classes needed.

.grid {
  row-gap:    var(--scaffold-row-gap, var(--scaffold-gap, 1rem));
  column-gap: var(--scaffold-col-gap, var(--scaffold-gap, 1rem));
}

<!-- Tighter rows, wider columns -->
<div class="scaffold-grid" style="--scaffold-row-gap: 0.5rem; --scaffold-col-gap: 2rem">
  ...
</div>

Breakpoints

Breakpoints are exposed as a SCSS mixin. Responsive utility classes are generated from that mixin and use a mobile-first prefix convention.

Default breakpoints

$breakpoints: (
  sm:  640px,
  md:  768px,
  lg:  1024px,
  xl:  1280px,
);

SCSS mixin

The mixin supports both directions — mobile-first (up) and desktop-first (down) — for use in your own stylesheets. Neither argument requires quotes.

@mixin breakpoint($size, $direction: up) {
  @if $direction == up {
    @media (min-width: map.get($breakpoints, $size)) {
      @content;
    }
  } @else {
    @media (max-width: map.get($breakpoints, $size) - 1px) {
      @content;
    }
  }
}

// Usage in your own SCSS
.my-component {
  font-size: 14px;

  @include breakpoint(md) {
    font-size: 16px;
  }

  @include breakpoint(lg, down) {
    display: none;
  }
}

Responsive utility classes

Responsive variants are generated for span classes and flex direction classes. The unprefixed class is the default (no media query); prefixed classes apply at that breakpoint and above.

<!-- Full width by default, half at md and up -->
<div class="scaffold-grid-span-full md:scaffold-grid-span-half">...</div>

<!-- Stacked on mobile, row at md and up -->
<div class="scaffold-flex scaffold-flex-col md:scaffold-flex">...</div>

Span variants cover every span × every breakpoint prefix. Flex direction variants cover scaffold-flex, scaffold-flex-col, and scaffold-flex-nowrap × every breakpoint prefix. Alignment classes are not generated responsively — use the breakpoint() mixin for those cases. Adding or removing a breakpoint from $breakpoints automatically adjusts both generated sets.

Mixins

Every utility class in Scaffold is generated from a corresponding SCSS mixin. Use mixins when you want Scaffold's layout logic inside your own semantic class names, without adding Scaffold classes to your HTML.

Import mixins via the SCSS module system:

@use 'scaffold';

Mixin argument vs custom property override

For any mixin that accepts arguments, there are two ways to customize the output — a mixin argument or a CSS custom property — and they suit different situations.

A mixin argument is a compile-time decision. The value gets baked into the generated CSS and is fixed. Use this when the layout is part of your design system and will never change at runtime — a sidebar that's always 4 columns, a card grid that's always 3.

// Baked in at compile time — --scaffold-cols cannot override this instance
.sidebar { @include scaffold.grid($cols: repeat(4, 1fr)); }

A custom property override is a runtime decision. The value stays live in the browser and can be changed per-instance in HTML or overridden by a parent scope. Use this when you need flexibility — a layout whose structure varies by context, or one you want to theme from outside the stylesheet.

<!-- Overridden at runtime — flexible, themeable -->
<div class="scaffold-grid" style="--scaffold-cols: repeat(4, 1fr)">...</div>

grid() mixin

Sets up a 12-column grid container.

@mixin grid(
  $cols: var(--scaffold-cols, repeat(12, 1fr)),
  $gap:  var(--scaffold-gap, 1rem)
) { ... }

// Examples
.page-grid    { @include scaffold.grid(); }
.sidebar-grid { @include scaffold.grid($cols: repeat(4, 1fr)); }
.tight-grid   { @include scaffold.grid($gap: 0.5rem); }

span() mixin

Sets grid-column on a grid child. Accepts the same semantic tokens as the utility classes.

@mixin span($size) { ... }

// Accepted values: full | half | third | two-thirds | quarter | three-quarters
.article   { @include scaffold.span(two-thirds); }
.sidebar   { @include scaffold.span(third); }
.hero      { @include scaffold.span(full); }

flex() mixin

Sets up a flex container. All arguments are optional; defaults match the .scaffold-flex utility class.

@mixin flex(
  $direction: row,
  $wrap:      wrap,
  $gap:       var(--scaffold-gap, 1rem)
) { ... }

// Examples
.nav-bar   { @include scaffold.flex(); }
.card-stack { @include scaffold.flex($direction: column, $wrap: nowrap); }

container() mixin

Creates a centered max-width wrapper. Pass $full: true for the full-width variant.

@mixin container(
  $width:      var(--scaffold-container-width, 1280px),
  $padding:    var(--scaffold-container-padding, 1.5rem),
  $padding-sm: var(--scaffold-container-padding-sm, 1rem),
  $full:       false
) { ... }

// Examples
.page         { @include scaffold.container(); }
.page--narrow { @include scaffold.container($width: 720px); }
.page--full   { @include scaffold.container($full: true); }

breakpoint() mixin

Wraps content in a media query. Supports both mobile-first (up, default) and desktop-first (down) directions.

@mixin breakpoint($size, $direction: up) { ... }

// Accepted sizes: sm | md | lg | xl
.article {
  @include scaffold.span(full);

  @include scaffold.breakpoint(md) {
    @include scaffold.span(two-thirds);
  }
}

.sidebar {
  @include scaffold.breakpoint(md, down) {
    display: none;
  }
}
